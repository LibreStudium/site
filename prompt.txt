Ты работаешь как senior frontend/UX-разработчик. Нужно сделать предпрелизную landing-страницу для будущего учебного веб-портала LibreLMS на Astro.

Контекст проекта:
Мы разрабатываем информационный веб-портал и элементы бренда ИТ-экосистемы LibreLMS.

Краткое описание:
«Разработка информационного веб-портала и элементов бренда ИТ-экосистемы LibreLMS.

Разработка веб-портала экосистемы LibreLMS, включающего модуль новостей, документации, инструментов дистрибуции ПО и руководства по стилю бренда.»

Текущая задача:
Нужно сделать временную предпрелизную страницу для домена. Когда пользователь заходит на сайт, он должен увидеть аккуратный минималистичный экран:
1. актуальный логотип LibreStudium/LibreLMS;
2. плавную анимацию появления логотипа;
3. текст о том, что портал скоро откроется;
4. ощущение технологичного, образовательного и аккуратного бренда;
5. адаптивную вёрстку под desktop/mobile;
6. готовность к хостингу через Onreza.

Важно:
- Внимательно изучи файл `LibreStudium Brandbook (standalone).html`.
- Используй из брендбука стиль, цветовую палитру, типографику и общую визуальную логику.
- Старое лого из брендбука не использовать как актуальное.
- Актуальные логотипы лежат отдельными PNG-файлами. Используй их.
- Не придумывай лишний функционал портала. Сейчас нужен только pre-release / coming soon экран.
- Не делай полноценный LMS-интерфейс, личный кабинет, новости, документацию и т. д. Это будет позже.
- Код должен быть чистым, понятным и готовым к дальнейшему расширению проекта.

Брендовые ориентиры из брендбука:
- Основной цвет: Deep Blue `#1E3A8A`.
- Акцентный цвет: Bright Blue `#3B82F6`.
- Светлый акцент: Light Blue `#93C5FD`.
- Цвет текста: Ink `#0F172A`.
- Светлый фон/поверхности: Mist `#EEF1F5`.
- Белый фон: `#FFFFFF`.
- Типографика: Jost для заголовков/брендовых элементов, Manrope для основного текста.
- Общий стиль: современный образовательный IT-бренд, чистый интерфейс, свободная электронная информационно-образовательная среда.

Технологии:
- Astro.
- TypeScript, если он уже включён в шаблон.
- CSS Modules или обычный CSS внутри Astro-компонентов — выбрать самый простой и поддерживаемый вариант.
- Без тяжёлых UI-библиотек.
- Анимации через CSS, без GSAP/Framer Motion.
- Не использовать React/Vue/Svelte без необходимости.
- Проект должен собираться командой `npm run build`.

Что нужно реализовать:
1. Создать базовую Astro-структуру.
2. Главная страница `/`:
   - fullscreen hero section;
   - по центру актуальный логотип;
   - ниже заголовок:
     `LibreLMS скоро откроется`
   - подзаголовок:
     `Мы готовим информационный веб-портал ИТ-экосистемы LibreLMS: новости, документацию, инструменты дистрибуции ПО и руководство по стилю бренда.`
   - маленькая строка статуса:
     `Предрелизная версия · запуск в подготовке`
3. Добавить плавную анимацию:
   - появление логотипа;
   - лёгкий scale/fade-in;
   - мягкое свечение или градиентный фон, но без перегруза.
4. Добавить favicon на основе актуального логотипа, если возможно.
5. Добавить SEO/meta:
   - title: `LibreLMS — скоро открытие`
   - description: `Информационный веб-портал ИТ-экосистемы LibreLMS. Новости, документация, инструменты дистрибуции ПО и руководство по стилю бренда.`
   - lang: `ru`
6. Добавить responsive-верстку:
   - на мобильных логотип не должен быть слишком большим;
   - текст должен быть читаемым;
   - отступы должны быть аккуратными.
7. Подготовить проект к статическому деплою:
   - проверить `astro.config.mjs`;
   - убедиться, что сайт собирается в `dist`;
   - не использовать серверные функции, если они не нужны.
8. Добавить README.md:
   - как установить зависимости;
   - как запустить dev-сервер;
   - как собрать проект;
   - что проект рассчитан на статический хостинг через Onreza.

Ожидаемая структура:
- `src/pages/index.astro`
- `src/components/ComingSoonHero.astro`
- `src/layouts/BaseLayout.astro`
- `src/styles/global.css`
- `public/logo/` — актуальные логотипы

Требования к качеству:
- Не оставляй мусорный код.
- Не используй временные lorem ipsum.
- Не добавляй ненужные страницы.
- Не подключай лишние зависимости.
- Перед финалом проверь сборку.
- Если чего-то не хватает в файлах, прямо напиши, чего именно не хватает.
- Если данные из брендбука конфликтуют с актуальными логотипами, приоритет у актуальных PNG-логотипов.
